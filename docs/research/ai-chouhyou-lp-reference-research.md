## 1. 調査目的

AI帳票番頭LPを「問い合わせが増えるLP」に作り直す前段階として、GitHub Pages 等で公開されている参考LPの**構成・信頼演出・CTA設計**を調査した。

売る本体はテンプレではなく、次の導入支援サービスである。

> 現場業者の散らかった受付・請求・売上管理を、現場経験17年の照屋が一緒に整理し、AIも使いながら実際に回る形へ整える導入支援サービス。

そのため、参考対象は「士業・個人コンサル・伴走支援・小規模事業者向けバックオフィス支援」に寄せ、**AIツール販売・テンプレ販売・情報商材・大企業SaaS LP**は除外した。

## 4. 共通して良かった構成

実務寄り・相談獲得寄りのLPに共通していたパターン。

1. **顔・肩書き・実務経験が早い**  
   士業実サイト（麻生税務会計事務所）はコンサルタント紹介が上部。個人の信頼がサービス信頼より先。

2. **一言約束が「誰の何をどうするか」**  
   Circustは「事業の循環を仕組みに」。士業テンプレは「手続きの、その先の安心まで」。抽象SaaS文言より具体的。

3. **初回無料・相談だけOKが明示**  
   複数サイトで FV または直後に配置。料金の不安より「まず話せる」安心が先。

4. **共感 → 解決 → 人 → 料金 → CTA**  
   Circust・士業テンプレ・業務改善コンサルで近い。機能一覧が先に来ない。

5. **料金は「何の作業の対価か」が分かる**  
   士業テンプレの表形式、Circustの無料＋スポット＋顧問の段階が分かりやすい。

6. **FAQは問い合わせ不安に絞る**  
   「何を相談していいか分からない」「個人情報」「断ってよいか」系。機能説明のFAQではない。

7. **セクション数は多くても、各ブロックの役割が単一**  
   良い例は1セクション1メッセージ。悪い例は1画面に訴求が詰まりすぎ。

8. **CTA文言が「診断・購入」より「相談・話す」**  
   「無料相談を予約」「お問い合わせ」が中心。

## 5. AI帳票番頭に取り入れるべき要素

### ファーストビュー

- **照屋の顔写真＋「現場業17年」＋一人親方向け**を左（または上）に配置
- キャッチは長文OKだが、**1文目で「散らばった案件情報を整える」**、2文目で「照屋が一緒に」
- CTAは主1つ: **「今の受付・請求まわりを無料で相談する」**
- 副CTAはページ内スクロール（共感セクション）程度に抑える
- FV直下に **相談だけOK / 個人情報は隠してOK / 強引な営業なし** の3点

### 信頼演出

- ヒーローまたはその直後に **自社現場で運用している** 事実（BCサービス）
- プロフィールは「システム会社ではない・今も現場に出ている」を太字級で
- 士業サイトのように **得意業種を列挙**（エアコン清掃・設備・便利屋等）
- お客様の声が無い現状では、**経験ベースの共感**（月末に探す・LINEに残る）で補強

### 構成・情報量

- セクション目安 **7〜9**（下記「推奨構成」参照）
- Before/Afterは **「いま / 整えたあと」** の2列で1画面以内
- 「一緒に整える内容」はカード6枚まで。各カードは**業務設計の言葉**（テンプレ名を出さない）
- デモ画像・管理シート画像は **ページ下部の補足**。「売っている商品」に見せない

### CTA・料金

- LINE CTA前に毎回 **安心1行**（相談だけOK等）
- 料金プランのリード文で **「整理・設計・導入支援の料金」** と明記
- 無料診断は **¥0・相談だけで終了可** を料金表の入口として強調
- 士業テンプレのように **相談の流れ3ステップ** を料金直前に置く

### デザイン

- **白＋余白＋ネイビー締め**。グラデーション主体は避ける
- フォントサイズは現状より**やや小さく・行間広め**（誠実・読みやすさ）
- カードは使うが、**アイコンだらけのSaaS風グリッド**は避ける
- 固定フッターCTAはスマホのみ、文言は短く「無料で相談（LINE）」

## 6. 取り入れない方がいい要素

| 避ける要素 | 理由 |
|------------|------|
| 「AIとテンプレで」「〇〇テンプレを渡す」前面訴求 | テンプレ販売・ツール販売に見える（現LPの課題） |
| 青グラデーション＋機能フロー図のヒーロー | 無料SaaS・簡易ツールLP印象 |
| 6〜8個の機能カードを並べるだけの中段 | 中身のないプロダクトLPに見える |
| Before/Afterの「テンプレですぐ作成」系コピー | 成果物販売の期待を煽りすぎる |
| 架空の実績・口コミ・相談件数 | 信頼を損なう |
| ThemeWagon系の英語テンプレ構成 | 日本の現場業者に刺さらない |
| dark theme・紫グラデのビジネステンプレ | 高額コンサル・情報商材寄り |
| 「完全自動化」「AIが全部やる」 | AI帳票番頭の安全設計（人が確認）と矛盾 |
| 長い用語解説ブロック（Make/Airtable等）の前出し | 難しいDX印象。必要ならFAQ末尾 |
| 料金を「テンプレ代」に見える見出し・箇条書き | 誤解を招く |

| 順 | セクション | 目的 | 参考元の要素 |
|----|------------|------|--------------|
| 1 | **ファーストビュー** | 誰向け・何を・誰が・無料相談 | 麻生税務（顔・業種）、Circust（無料明示） |
| 2 | **信頼バンド**（4項目） | 現場17年・自社運用・相談だけOK・個人情報OK | 士業テンプレのFVバッジ |

| 5 | **なぜ照屋が分かるのか** | 自社運用例＋プロフィールを1ブロックに統合 | 麻生税務のコンサルタント紹介 |

| 9 | **整理イメージ（画像）** | デモ4枚。補足扱い・キャプションは「参考」 | 現LPのdemoを下部へ |
| 10 | **最終CTA** | LINE1本＋安心3点 | CircustのCONTACT |

**削除・統合候補（現LPから）**

- 「選ばれる理由」「向いている方」を独立セクションにせず、信頼・FAQに分散
- 関連コラムはフッター直前でよいが、**問い合わせ導線より上に置かない**

## 付録: 各サイトの観点別メモ

### 麻生税務会計事務所（asaokaikei/mie-suzuka）

- **FV**: サイト全体が事務所サイト型（LP単页ではない）
- **信頼**: 顔写真・略歴・建設業等の業種実績・M&A登録など権威は自然
- **CTA**: フォーム＋初回無料明記
- **AI帳票番頭への示唆**: プロフィールの厚みと業種明示は真似る。ページ長さは真似しない

### Circulation Structure（circust328-png/circust.com）

- **FV**: キャッチ＋無料相談CTA＋数字4つ（サービス領域・売上規模・初回無料・全国）
- **信頼**: MBA・実務・地域密着（人物像はやや後ろ）
- **CTA**: 無料相談が複数回出るが、主文言は統一
- **AI帳票番頭への示唆**: 共感6項目→解決の流れは参考になる。規模感・デザインのリッチさは削る

### cocoroai/profession-office-template

- **FV**: 短いキャッチ＋2CTA＋信頼3数字
- **構成**: REASON → SERVICES → FLOW → FEES → PROFILE → FAQ（士業LPの教科書）
- **AI帳票番頭への示唆**: セクション順と料金表の見せ方のみ。架空事務所のためデザイン・数字は流用不可

### 業務改善コンサル（coco556）

- **FV**: 「面倒な業務を効率化」→ ややツール/SaaS寄り
- **良い点**: 共感・Before/After事例
- **AI帳票番頭への示唆**: 事例の型のみ。自動化・システム構築の主語は使わない

### ポラリス経理（polariskeiri）

- **信頼**: お客様の声・サービスの流れ・提携先
- **AI帳票番頭への示唆**: 「記帳代行の流れ」＝「無料相談の流れ」に置き換え可能。声が無ければ作らない
